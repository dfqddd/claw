## 🎯 技能目标

从股市论坛（淘股吧、东财等）收集**经过实盘验证**的交易模式，甄别筛选后总结成可复制的交易系统。

### 核心价值

```
市场已经证明的 > 自己凭空设计的
实盘验证的 > 理论推导的
长期稳定的 > 短期暴利的
```

## 🚀 使用流程

### 步骤 1：收集实盘数据

```bash
cd ~/.openclaw/workspace-hanli/skills/trading-pattern-miner

# 爬取淘股吧实盘帖
python3 scripts/crawl_taoguba.py

# 爬取东财实盘帖（待实现）
python3 scripts/crawl_eastmoney.py

# 爬取雪球组合（待实现）
python3 scripts/crawl_xueqiu.py
```

**输出**: `data/taoguba_traders.json`

---

### 步骤 2：甄别筛选

```bash
# 自动筛选（有交割单 + 连续更新）
python3 scripts/filter_traders.py
```

**筛选标准**:
```
✅ 保留:
  - 有交割单截图
  - 连续更新>30 天
  - 回复数>100（有人关注）
  - 收益率合理（月收益<50%）

❌ 排除:
  - 只有理论没有实盘
  - 断更>7 天
  - 只报喜不报忧
  - 推荐股票收费
```

**输出**: `data/verified_traders.json`

---

### 步骤 3：提取交易记录

```bash
# 从实盘帖中提取交易记录
python3 scripts/extract_trades.py
```

**提取内容**:
```json
{
  "trader_id": "淘股吧 - 作手新一",
  "date": "2026-03-05",
  "code": "000001",
  "name": "平安银行",
  "action": "买入",
  "price": 10.50,
  "shares": 10000,
  "amount": 105000,
  "reason": "突破 20 日均线，放量",
  "profit_pct": null
}
```

**输出**: `data/all_trades.json`

---

### 步骤 4：总结交易模式

```bash
# 分析交易记录，提取模式

```

**输出示例**:

## 📊 模式分类

### 按操作方式

| 类型 | 说明 | 代表模式 |
|------|------|---------|
| **打板** | 涨停板买入 | 首板、连板、反包板 |
| **低吸** | 支撑位埋伏 | 均线低吸、平台低吸 |
| **半路** | 启动初期介入 | 突破半路、加速半路 |
| **龙头** | 做最强股 | 龙空龙、龙头首阴 |
| **套利** | 价差交易 | 板块套利、可转债 |

### 按持仓周期

| 类型 | 周期 | 特点 |
|------|------|------|
| **超短线** | 1-2 天 | 快进快出，积小胜为大胜 |
| **短线** | 3-5 天 | 吃主升浪，5-10% 止盈 |
| **中线** | 5-20 天 | 趋势跟踪，20%+ 止盈 |
| **长线** | 20 天 + | 价值投资，翻倍目标 |

## ✅ 模式有效性标准

### 有效模式（可实盘）

```
✅ 样本数 > 50 次交易
✅ 胜率 > 50%
✅ 盈亏比 > 2:1
✅ 最大回撤 < 20%
✅ 至少 3 个选手使用
✅ 连续 6 个月有效
```

### 待验证模式（观察）

```
⚠️ 样本数 20-50 次
⚠️ 胜率 45-50%
⚠️ 需要更多数据验证
```

### 无效模式（排除）

```
❌ 样本数 < 20 次
❌ 胜率 < 45%
❌ 盈亏比 < 1:1
❌ 最大回撤 > 30%
❌ 只有理论无实盘
```

## 🛡️ 质量控制

### 实盘验证等级

```
Level 1: ⭐ 有交割单截图
Level 2: ⭐⭐ 交割单连续 3 个月
Level 3: ⭐⭐⭐ 收益率与大盘对比合理
Level 4: ⭐⭐⭐⭐ 买卖逻辑一致
Level 5: ⭐⭐⭐⭐⭐ 模式可复制
```

**只采用 Level 3+ 的模式**

### 数据交叉验证

```
1. 帖子内容 vs 交割单 → 是否一致
2. 收益率 vs 大盘 → 是否合理
3. 多个选手 → 是否趋同
4. 历史回测 → 是否有效
```

## ⚠️ 注意事项

### 法律风险
```
1. 仅用于个人学习研究
2. 不用于商业用途
3. 尊重原作者知识产权
4. 不传播收费内容
```

### 数据真实性
```
1. 交割单可能造假 → 交叉验证
2. 收益率可能夸大 → 对比大盘
3. 只报喜不报忧 → 找完整记录
4. 幸存者偏差 → 看失败案例
```

### 模式适用性
```
1. 市场环境变化 → 模式可能失效
2. 资金量影响 → 大资金难复制
3. 个人风格差异 → 适合的不一定适合你
4. 需要实践验证 → 小仓位测试
```
